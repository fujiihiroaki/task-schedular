## Why

現在のペースタグは `slow`（120日）、`normal`（90日）、`fast`（30日）の3段階しかなく、ユーザーがより細かい粒度でリードタイムを制御したい場合に対応できない。また、新しいペースレベルを追加するには直接コードを変更する必要がある。ペースタグの拡充と日数のカスタマイズにより、ユーザーの多様なタスク管理ニーズに対応する。

## What Changes

- 新しいペースレベルの追加。
  - pace:nonstop = 1日前から着手
  - pace:rapid = 7日前から着手
- 既存のペースレベル（`slow`, `normal`, `fast`）のデフォルト日数を変更する
  - pace:slow = 90日前から着手
  - pace:normal = 30日前から着手
  - pace:fast = 14日前から着手
- `pace:<name>=<days>` 形式でユーザーが任意の日数を直接指定できるようにする
  - `<days>` は `1` 以上 `2048` 以下の整数のみ許可する
  - ペースタグ名（`<name>`）は大文字小文字を区別しない
  - 同一タスク内で `pace` 指定が重複した場合は、後で読んだ指定を採用する
- パーサーが新しいペースタグ形式を認識・解析できるようにする
- 不正なペース値に対するバリデーションの追加
  - 不正値を検出した場合はエラーメッセージを表示し、その `pace` 指定は無視する
- `start` / `lead` / `pace` の優先順位は現行仕様を維持する
  - `start` 明示指定を最優先する
  - `start` 未指定時に `pace` があれば `lead` を上書きする
  - `pace` が有効でない場合は既存の `lead` 推論にフォールバックする（タグ/キーワード不一致時の既定値は 1 日）
- 既存ユーザーへの移行考慮は行わない（既存の挙動変更影響は本変更で扱わない）
- README を更新し、ペースタグの日数表を新しい既定値に修正し、`nonstop` / `rapid` を追記する
- ペースタグは正規化して出力する
  - 出力時のペースタグ名は小文字に統一する
  - 既定ペースは `pace:<name>` 形式で出力する
  - カスタム日数付きペースは `pace:<name>=<days>` 形式で出力する（`<days>` は整数表記）

## Capabilities

### New Capabilities
- `pace-tag`: ペースタグの定義、パース、日数変換、バリデーションを含むペースタグ機能全体の仕様

### Modified Capabilities
（なし — 既存の spec はペースタグを直接扱っていないため）

## Impact

- `TaskSchedular/MarkdownTaskParser.cs`: ペースタグの正規表現パターンと解析ロジックの拡張
- `TaskSchedular/Inference.cs`: `_PaceToLead()` メソッドの拡張（新ペースレベル対応・カスタム日数対応）
- `TaskSchedular/TaskItem.cs`: `Pace` プロパティの型または保持形式の変更の可能性
- `TaskSchedular/MarkdownRenderer.cs`: 新形式のペースタグのレンダリング対応
- テストファイル: 新ペースレベル・カスタム日数のテストケース追加
