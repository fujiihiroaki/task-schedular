## ADDED Requirements

### Requirement: 入力ファイルに期間を定義できる
システムは入力ファイルの先頭領域に記述された `period: YYYY-MM-DD .. YYYY-MM-DD` 形式の行を解析し、期間の開始日と終了日を認識しなければならない（MUST）。

#### Scenario: 有効な period 行を解析する
- **WHEN** 入力ファイルに `period: 2026-01-01 .. 2026-03-31` と記述されている
- **THEN** システムは開始日を `2026-01-01`、終了日を `2026-03-31` として認識する

#### Scenario: period 行が存在しない場合
- **WHEN** 入力ファイルに `period:` 行が存在しない
- **THEN** システムは期間未定義として扱い、既存の動作を変更しない

#### Scenario: period 行が複数存在する場合
- **WHEN** 入力ファイルに複数の `period:` 行が記述されている
- **THEN** システムは最後に記述された `period:` 行の値を採用する

### Requirement: 今期セクション内タスクに期末日を締切として自動設定する
`period:` が定義されている場合、セクション名が「今期の」で始まるセクション内の未完了タスクに対して、期間の終了日を `due` として自動設定しなければならない（MUST）。

#### Scenario: 今期セクション内のタスクに due を自動設定する
- **WHEN** `period: 2026-01-01 .. 2026-03-31` が定義されている
- **AND** セクション名が「今期のバックログ」である
- **AND** タスクに `due:` が指定されていない
- **THEN** システムはそのタスクの `due` を `2026-03-31` に設定する

#### Scenario: 明示的な due 指定を上書きしない
- **WHEN** `period: 2026-01-01 .. 2026-03-31` が定義されている
- **AND** セクション名が「今期のバックログ」である
- **AND** タスクに `due: 2026-02-28` が明示的に指定されている
- **THEN** システムはそのタスクの `due` を `2026-02-28` のまま維持する

#### Scenario: 今期以外のセクションには適用しない
- **WHEN** `period: 2026-01-01 .. 2026-03-31` が定義されている
- **AND** セクション名が「今期の」で始まらない（例:「来月のタスク」）
- **AND** タスクに `due:` が指定されていない
- **THEN** システムはそのタスクの `due` を自動設定しない

#### Scenario: period 未定義の場合は今期セクションでも適用しない
- **WHEN** 入力ファイルに `period:` 行が存在しない
- **AND** セクション名が「今期のバックログ」である
- **THEN** システムはそのタスクの `due` を自動設定しない
